팀 프로젝트

NOON

건물, 장소를 기반으로 서로 소통하고 연결되는 소셜 네트워크 서비스

프로젝트 설명 (개발목표)

사용자가 건물을 등록하고, 등록한 건물을 소통의 장으로 만든다. 건물을 등록하고 소통함으로써 일상공간을 모델로 한 새로운 SNS를 만들고자 하였다.

프로젝트 기간

2024년 5월 ~ 2024년 7월 (분석 · 설계 5주 / 구현 4주)

개발환경

  • IntelliJ 및 VS Code에서 개발
  • 백엔드 애플리케이션은 Java 기반 Spring Framework 적용
  • 프론트엔드 애플리케이션은 JavaScript 기반 React를 적용
  • 채팅 서버는 JavaScript 기반 Node.js + Express 적용
  • 내장 톰캣을 사용해 REST API 서버 배포
  • NGINX를 위에 웹 서버 배포
  • 컨테이너 환경에서 애플리케이션을 배포하며 Kubernetes를 사용해 컨테이너 관리

담당업무

  • 팀장 직책
  • 지도 시스템 구현
  • 환경설정 시스템 구현
  • 건물위키 시스템 구현
  • 공통모듈 구현
  • Jenkins를 활용한 CI/CD Pipeline 구축
  • Spring Profile 활용
  • 환경에 따라 다른 메타데이터 활용
  • 환경에 따라 다른 Spring Bean 등록
  • Dockerfile 작성을 통한 웹 애플리케이션 컨테이너 이미지 등록
  • kubeadm을 통한 Kubernetes 클러스터 구축
  • NAVER Certificate Manager를 통해 애플리케이션 로드밸런서에 SSL 인증서 등록
  • NAVER CLOVA Chatbot을 통해 챗봇 구현
  • 오토스케일링 시스템 구축
  • 서버 이미지와 Init Script를 활용해 Scale-Out된 서버 자동으로 Kubernetes 클러스터에 워커 노드로 등록
  • Horizontal Pod Autoscaling (HPA)를 활용한 Pod 오토스케일링
  • Ajax를 활용해 사용자 타이핑에 반응하여 검색 결과 미리 가져오기

구현한 공통모듈

  • Spring Security 기반 인증 · 인가 로직
  • Security Filter
  • JWT 기반 토큰 인증
  • OAuth 2.0 기반 외부 로그인 토큰 인증
  • 사용자의 권한에 따른 리소스 접근 제어
  • Java Reflection API를 활용한 Validator
  • Service Layer로부터 검증 로직 분리
